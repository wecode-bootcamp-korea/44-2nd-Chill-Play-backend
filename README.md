## About 'muse'
![image](https://user-images.githubusercontent.com/123849268/236437851-da1ea13f-79b3-4d12-8044-3212fcf81553.png)
<br>뮤지컬은 대중적으로 많은 사람들이 관심을 가지고 있는 예술의 한 분야입니다. 관람객의 오감을 자극하고 힐링과 영감을 준다는 의미에서
저희 웹서비스를 '뮤즈'라고 지었습니다. <br>'뮤즈'를 통해 인기 있는 작품을 추천받고, 빠르고 간편한 절차를 통해 뮤지컬을 예매할 수 있습니다.

## 회원가입/로그인
![login](https://user-images.githubusercontent.com/123074891/236431888-e2b89c1a-64f3-40b4-99d8-8b1ec0ea0c87.gif)
<br>
**카카오 소셜 로그인 API 활용** <br>
저희는 로그인 과정에서 소셜 로그인 기능을 활용하였습니다.
대중적으로 많은 사람들이 이용하는 카카오를 로그인 채널로 제한하여, 보다 많은 사용자 유치 및 서비스 접근의 편의성을 확대함으로써, 보다 쉽게 뮤지컬 예매가 가능하도록 구현하였습니다.
또한, 로그인 과정에서 카카오와의 통신을 통해 카카오 인가 토큰을 받음으로써, 사용자에 대한 정보 저장과 서비스 이용에 필요한 권한을 부여하였습니다.

[FE]
  - 대중적으로 많이 사용하는 카카오계정 소셜 로그인을 활용. 복잡하지 않은 로그인 과정을 통해 사용자의 편의성을 증대하여 보다 쉽게 뮤지컬을 예매할 수 있도록 구성
	
[BE]
  - 로그인시, 전달받은 카카오 인가토큰을 카카오와 다시 통신하여 유저에 대한 정보를 전달받음
  - 카카오에서 전달받은 유저 정보는 user 테이블에 데이터가 존재하지 않으면, DB에 회원정보 저장 이후 JWT토큰을 클라이언트에게 발행

## 랜딩페이지
![main](https://user-images.githubusercontent.com/123074891/236431916-9672f987-55ff-4a23-85ef-1a074909b401.gif)
상세나 예매하기 페이지로 이동 가능한 뮤지컬 상위 차트와 상영중인 뮤지컬 홍보를 위한 관련 영상을 상단 배너에 배치하고, 서비스 이용시 필요로하는 상영일자, 예몌율, 관람 등급 등 다양한 정보를 한번에 파악 할 수 도록 구성하였습니다. 또한, 스크롤 반응을 통해 페이지 하단부에서 사용자에게 다양한 컨텐츠를 제공 할 수 있도록 레이아웃을 구성했습니다.

[FE]
	<br>- 사용자가 처음 사이트를 진입할 때 보여지는 화면으로 스크롤에 반응하여 컨텐츠가 나타나도록 구성. <br>
	- 상세나 예매하기 페이지로 바로 이동할 수 있는 뮤지컬 상위차트와 상영중인 뮤지컬을 홍보하기 위해 상단에 배너 배치.

[BE]
  - 뮤지컬 관련 상영일자, 예매율, 이미지 등 관련 정보 전송.
<br>
<br>

## 네비게이션
![Nav-min](https://user-images.githubusercontent.com/123074891/236439607-d7bc5137-cf34-46d0-ad68-68fc3a8413a9.gif)
메인페이지 상단에 위치한 네비게이션과 검색 창 기능입니다. 메인 페이지의 상단부를 보시면 네비게이션바와 검색 기능 창이 있는 것을 확인 하실 수 있습니다.
사용자의 서비스 이용에 편의성을 제공하고자 뮤지컬, 극장, 예매 별로 구분되고, 접근하기 쉽도록 네비이션바를 구성하였습니다.
<br>전체 리스트를 둘러보지 않아도, 검색 창에 부여된 LIMIT 값을 통해 사용자에게 전체 뮤지컬 리스트를 노출하여 파악 하기 쉽도록 의도하였습니다. 또한, 검색창에 입력되는 키워드에 따라 해당 뮤지컬 정보를 출력하여 빠르게 상세 페이지로 이동하도록 구현하였습니다.

[FE]
  - 검색에 따라 도출되는 결과 출력
  - 전체 리스트를 둘러보지 않아도 검색 창에서 빠르게 해당 뮤지컬 상세 페이지로 이동하도록 구성

[BE]
  - Limit 값을 부여하여 전체 뮤지컬 리스트의 일부를 검색 창에 출력. 
  - 뮤지컬 키워드 입력시, 해당 키워드에 대한 뮤지컬 정보를 띄우도록 정보를 반환함.
<br>
<br>

## 뮤지컬 리스트 페이지
![Muse-List](https://user-images.githubusercontent.com/123074891/236431942-f78fc10d-af27-4f39-b0b4-48d8f946fd93.gif)
필터 기능을 추가하여, 뮤지컬 총 판매량을 기반으로한 예매율, 개봉일자, 관람등급의 조건을 걸어 
사용자가 원하는 조건에 알맞는 뮤지컬을 확인 할 수 있도록 구현하였습니다. 또한 상영 예정작만 볼 수 있는 기능을 통해 
추후 어떤 작품이 상영될 것인에 대한 정보도 제공하고 있습니다. 
<br>마찬가지로 리스트 페이지에서도 각 뮤지컬에 대한 예매율, 개봉일자, 관람 등급 정보를 한눈에 파악하기 쉽도록 구성하여
사용자의 편의성을 강조하였습니다.

[FE]
  - 자신이 원하는 조건에 따라 뮤지컬을 추천받을 수 있다.

[BE]
  - 뮤지컬의 총 판매량을 기반으로 한 예매율, 개봉일자, 관람등급을 사용자가 원하는 조건에맞는 뮤지컬을 확인할 수 있게 필터 구현
<br>

## 뮤지컬 디테일 페이지
![Muse-detail-min](https://user-images.githubusercontent.com/123074891/236437178-a47d8a5d-90d7-4bb5-9c15-331614c80b05.gif)
페이지 상단부에 해당 뮤지컬 홍보 영상을 노출하여, 역동적인 홈페이지 구성을 통해 사용자의 흥미와 관심을 유발할 수 있도록 레이아웃을 구성하였습니다.
<br>간단한 뮤지컬 설명 글과 출연진, 스틸컷 이미지를 제공하여 해당 뮤지컬에 대한 사용자의 궁금증을 빠르게 해소하고, 예매율 그래프를 통해 어떤 성별과 연령에서의 많은 수요가 있는지 파악 할 수 있는 정보를 제공하는 레이아웃을 구성하였습니다.이를 통해 보다 해당 뮤지컬에 대한 내용을 사용자가 한눈에 파악하기 쉽도록 하여 저희 제품을 통해 소비자의 니즈에 적합한 컨텐츠 선택에 도움이 되는 서비스를 제공하고 있습니다.

[FE]
 - 디테일 페이지에서는 사용자가 해당 뮤지컬에 대한 궁금증을 빠르게 해소 시키고, 예매율 그래프를 통해서 사용자의 성별, 나이대에 맞게 추천 받을 수 있도록 구성.

[BE] 
 - 뮤지컬 상세 정보 전달 및 예매율에 대해서 성별과 연령으로 파악 할 수 있게 표시하여 소비자의 뮤지컬 선택에 도움을 제공.
<br>
<br>

## 예매페이지, 좌석 선택페이지
![Muse-booking](https://user-images.githubusercontent.com/123074891/236432842-8707f250-90d0-4089-ab00-fbdc530d1825.gif)
실시간 예매율이 높은 상품을 나열하여,작품 선택시 어떤 작품의 수요가 많은지 파악 할 수 있게 배치하였습니다. 
<br>또한, 사용자가 예매 과정에 집중 할 수 있도록 작품선택, 장소, 상영날짜와 시간을 단계별 과정을 통해 선택 하도록 UI를 구성하였습니다. 이 과정에서 실시간으로 클릭한 아이템들에 대해 하이라이팅을 부여하여 사용자가 어떤 항목을 선택 하였는지 파악할 수 있도록 강조했습니다. 
<br>이에 추가적으로 예매 페이지 하단부에서 전체 주문 상태를 실시간으로 확인 할 수 있도록 하단의 검은 상태바를 구현하였습니다.
<br>상영일과 상영 시간을 선택하게 되면 이후 좌석 선택으로 이어지게 되고, 설정한 관람 인원수에 따라 VIP, 일반 좌석을 나누어서 선택 할 수 있도록 구성하였습니다.

[FE]
  - 예매에 집중할 수 있도록 유도한 UI, 
  - 사용자가 실시간으로 자신이 어떤 뮤지컬을 확인 할 수 있도록 선택한 아이템들에 하이라이팅 부여
  - 전체 주문 상태를 실시간으로 확인할 수 있도록 하단의 검은 상태바를 구현 (예매 하단의 예매 플로우 정보) 

[BE]
  - 클라이언트의 요청에 따라, 사용자에게 예매 선택권을 주기 위한 뮤지컬, 상영관, 날짜, 상영시간 정보를 전달.
  - 실시간 예매율이 높은 상품을 나열하며, 각 날짜의 기준에 맞게 시간표와 공연 좌석, 가격에 대한 정보를 전달. 
<br>
<br>

## 결제 페이지
![결제](https://user-images.githubusercontent.com/123074891/236432154-ffaf3225-9d09-4f20-a6cf-0da73f671057.gif)
Toss Payment 외부 api를 사용하여 보다 간단하고 편리한 결제가 진행 되도록 구현하였습니다. 클릭 몇번으로 결제가 완료되기 때문에, 결제 진행전 QR코드를 활용하여 사용자의 약관동의가 진행되어야 결제가 진행됨을 확인 하실 수 있습니다.
<br>이때, 예매 좌석정보와 예매율에 관련되는 판매량 정보에 트랜잭션을 적용하여, 예매 과정중 에러 발생시, 결제에 대한 정보가 없어지도록 설정하여 중복 결제를 방지하였습니다.

[FE]
  - Toss Payments 외부 API를 사용하여 사용자가 편리하게 결제를 진행 할 수 있도록 구현
  - QR코드를 활용하여 사용자의 결제 약관동의가 확인되어야 진행됨
     
[BE]
  - 예매 단계에서 저장된 데이터를 클라이언트로 부터 넘겨받아 예매 좌석정보와 예매율에 관련이 되는 판매량 정보를 트랜잭션을 주어서 에러가 발생시 결제에 대한 모두 정보가 없어질수있도록 설정
<br>
<br>

## MY페이지
![마이페이지 해상도 수정](https://user-images.githubusercontent.com/123074891/236436456-07c52dc8-f0f2-4af1-a2e6-e95a6bb0bff9.gif)
결제 이후, 사용자는 자신이 예약한 뮤지컬내역을 확인할 수 있어야 하기 때문에, 마이페이지에서 조회 할 수 있도록 구현하였습니다.
저희가 구현한 서비스가 티켓팅에 주력한 서비스인만큼, 가장 최근의 구매내역에 해당하는 정보를 실제 티켓과 유사하게 구현하여 사용자가 실제 티켓을 확인하는 것처럼 구성하였습니다. 
또한, 매번 번거롭게 사용자가 웹사이트를 들어와서 확인하지 않아도 티켓 정보를 알 수 있도록, 티켓을 별도로 저장하는 기능까지 구현하였습니다.

[FE]
  - 마이페이지를 통해 사용자의 예매 내역을 파악 할 수 있도록 구현.
  - 가장 최근의 구매내역에 해당하는 정보를 실제 티켓과 유사하게 구현.
